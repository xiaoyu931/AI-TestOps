🧠 12. 分析架构设计（Analyzer Core）
12.1 设计目标

系统中的错误分析逻辑从 API 层解耦，统一收口到分析核心模块（Analyzer Core），实现：

单一分析逻辑来源（Single Source of Truth）
规则与知识库统一管理
支持未来扩展（聚合、AI、反馈闭环）
12.2 当前问题

当前系统中：

错误分析逻辑写在 API 中（如 /failure-analysis）
存在规则多源（代码 + JSON + DB）
分析逻辑不可复用
12.3 目标架构
日志数据（DB）
        ↓
Error Extraction（标准化）
        ↓
Analyzer Core（统一分析层）
        ↓
规则匹配（error_rules）
        ↓
知识库映射（error_knowledge）
        ↓
分析结果
        ↓
API 层调用
        ↓
前端展示
12.4 分析流程定义
Step1：错误提取（component优先）
Step2：错误标准化（cleaning）
Step3：规则匹配（pattern + priority）
Step4：Root Cause 映射（knowledge）
Step5：聚合（Top Issue / 分布）
12.5 分析输出标准（统一结构）
{
  "error_type": "...",
  "root_cause": "...",
  "solution": "...",
  "category": "...",

👉 然后加一句（非常关键）：

所有分析类 API（failure_analysis、test_plan_health）必须基于该结构生成结果。
12.6 设计原则
分析逻辑不写在 API 中
规则来源统一为数据库
分析层可独立演进
API 只负责调用分析结果


**13. Analyzer Core 落地实现说明**

**13.1 实现目标**

本阶段完成 Analyzer Core 的第一阶段落地，目标是将错误分析逻辑从 API 层抽离，形成统一、可复用、可维护的分析核心。


**13.2 Batch 1：读 DB + Analyzer Core**

本阶段新增 Analyzer Core 的基础能力。

新增模块：

```text
backend/app/analyzer/
```

核心文件：

```text
analyzer_core.py
error_normalizer.py
rule_matcher.py
knowledge_mapper.py
confidence.py
```

新增数据库模型：

```text
backend/app/models/error_rule_model.py
backend/app/models/error_knowledge_model.py
```

职责说明：

```text
error_normalizer
负责清洗原始错误文本，生成可匹配文本。

rule_matcher
从 error_rules 表读取 active 规则，并按 priority 进行匹配。

knowledge_mapper
根据 error_type + root_cause 从 error_knowledge 表获取 solution、category、confidence。

confidence
将 high / medium / low 转换为内部数值评分。

analyzer_core
提供统一入口 analyze_error(db, error_text)。
```

Analyzer Core 标准输出：

```json
{
  "matched": true,
  "error_type": "Automation bug",
  "root_cause": "UI element location failed",
  "solution": "Check UI elements or update automation selectors",
  "category": "Automation",
  "confidence": "high",
  "confidence_score": 0.9,
  "matched_rule_id": 1,
  "matched_pattern": "selector",
  "source": "excel_cleaned",
  "knowledge_source": "excel_cleaned",
  "error_summary": "..."
}
```

未匹配时返回：

```json
{
  "matched": false,
  "error_type": "OTHER",
  "root_cause": "Unknown",
  "solution": "Review the detailed error log and classify this issue into a reusable rule if it happens repeatedly.",
  "category": "UNKNOWN",
  "confidence": "low",
  "confidence_score": 0.3
}
```

**13.6 当前实现后的模块边界**

